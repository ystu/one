# 大綱整理規則

## 一、最上方必須標明主題

每份大綱最上方必須先寫主題。

格式：

```md
主題：
```

規則：

1. 主題放在整份大綱最上方
2. 主題只寫本篇大綱的核心題目
3. 主題不使用標號
4. 主題後方可加冒號
5. 若要整理成一張 A4 提示版，主題下方不留空白行

範例：

```md
主題：
一、大標題
（一）次標題
1. 重點
2. 重點
3. 重點
```

## 二、標號層級規則

大綱可依用途使用兩層或三層標號。

第一層使用國字數字：

一、  
二、  
三、  
四、  
五、  

第二層使用國字括號：

（一）  
（二）  
（三）  
（四）  
（五）  

第三層使用阿拉伯數字：

1.  
2.  
3.  
4.  
5.  

使用原則：

1. 一般整理稿可使用三層標號
2. 一張 A4 提示版優先使用兩層標號
3. 兩層版只保留「一、」與「（一）」層級
4. 若第三層會造成版面過長，應把重點濃縮進次標題

## 三、第一層：大標題

第一層為大標題，使用「國字數字 + 頓號」。

格式：

```md
一、大標題
二、大標題
三、大標題
```

規則：

1. 大標題用來標示整個大段的主題
2. 大標題要能看出本段的方向
3. 大標題可以稍長，但不要寫成完整句子
4. 大標題之間要有順序推進
5. 不使用阿拉伯數字作為大標題

## 四、第二層：次標題

第二層為次標題，使用「全形括號 + 國字數字」。

格式：

```md
（一）次標題
（二）次標題
（三）次標題
```

規則：

1. 次標題用來拆解大標題下的重點
2. 次標題要比大標題更具體
3. 次標題要能看出本小段的功能
4. 次標題不宜過長
5. 不使用 1. 2. 3. 作為次標題

## 五、第三層：重點條列

第三層為具體重點，使用「阿拉伯數字 + 半形句點」。

格式：

```md
1. 重點
2. 重點
3. 重點
```

規則：

1. 每一點只寫一個重點
2. 內容要簡短明確
3. 不寫成完整講稿
4. 不堆疊太多抽象名詞
5. 每組條列以 3 到 5 點為原則
6. 若目標是一張 A4，第三層可省略
7. 省略第三層時，要把必要重點濃縮到次標題中

## 六、標題寫法規則

標題要有方向感，不只放名詞。

避免：

```md
一、定義
二、歷史
三、宗旨
```

建議：

```md
一、說明核心概念
二、建立背景脈絡
三、回到生活實踐
```

規則：

1. 標題要能看出段落目的
2. 標題要有帶領性
3. 標題不要太像資料分類
4. 標題不要寫得太口語零散
5. 同一層標題的語氣要一致

## 七、內容安排規則

大綱不是逐字稿，而是講述路線。

大綱與講稿的分工：

1. 大綱只保留骨架、次序、重點與提示
2. 講稿才負責提問、鋪陳、故事細節、情緒轉折與完整收束
3. 若一句話已經可以直接拿到台上講，通常要再濃縮
4. 「講述重點」「故事引導」「結語講法」不可寫成完整段落
5. 故事在大綱中只留「故事提示：人物或事件 + 核心作用」
6. 收束語在大綱中只留方向，不寫成完整結論句

每一大段可以依照以下順序整理：

1. 先提出問題
2. 再說明核心概念
3. 再補充依據或例子
4. 再回到實際應用
5. 最後收束本段重點

## 八、每段內部結構

每一段可用以下結構整理：

（一）提出問題

1. 聽眾可能有的疑惑
2. 生活中常見的情境
3. 本段要處理的核心問題

（二）說明道理

1. 本段主要觀念
2. 重要定義或說明
3. 必要的補充脈絡

（三）舉例或引證

1. 人物故事
2. 生活比喻
3. 經典、語錄或資料依據
4. 大綱只寫提示，不展開故事內容

（四）收回實踐

1. 回到自己身上
2. 落到生活行動
3. 收束本段重點
4. 大綱只寫收束方向，不寫講稿式結語

## 九、文字長度規則

大綱要簡明，不寫成講稿。

建議長度：

1. 大標題：10 到 25 字
2. 次標題：8 到 20 字
3. 重點條列：8 到 30 字
4. 每一大段不宜過度展開
5. 若內容已經像完整段落，應再濃縮
6. 避免出現連續兩句以上的講述文字
7. 避免把現場提問完整寫進大綱
8. 避免把故事背景、情節、轉折寫成段落

## 十、語氣規則

大綱語氣要清楚、穩定、有方向。

規則：

1. 用詞要簡潔
2. 避免過度修飾
3. 避免情緒太滿
4. 避免寫成口語逐字稿
5. 保留講稿發展空間

## 十一、輸出與排版規則

大綱先以 Markdown 格式整理，後續可轉成 Word 文件。

Markdown 格式：

1. 最上方必須標明主題
2. 第一層使用「一、二、三」
3. 第二層使用「（一）（二）（三）」
4. 一般版第三層可使用「1. 2. 3.」
5. 一張 A4 提示版可省略第三層
6. 一張 A4 提示版不留空白行
7. 保持層級清楚，不寫成逐字稿

Word 格式：

1. 字型使用標楷體
2. 字體大小使用 12
3. 行距使用單行距
4. 保留原本大綱層級
5. 不使用複雜樣式
6. 以清楚、穩定、容易閱讀為原則

Word 一張 A4 版：

1. 若使用者要求一張 A4，優先移除多餘空白行
2. 優先精簡內容層級，不要硬把文字擠在一起
3. 可拿掉第三層「1. 2. 3.」，改用兩層大綱
4. 可使用窄邊界壓縮篇幅
5. 保留標楷體與 12 號字，不用縮小字體硬塞
6. 段前段後距離可調小，但要保留閱讀空間
7. 原則上使用單欄；除非使用者明確要求，避免雙欄排版

## 十二、檔名命名規則

大綱檔案統一使用以下格式命名：

```md
{題目}大綱_凃信埕
```

規則：

1. `{題目}` 使用該篇大綱的主題名稱
2. 題目後直接接「大綱」
3. 「大綱」後接底線 `_`
4. 底線後接姓名「凃信埕」
5. 不額外加入日期、版本號或其他文字，除非另有要求

Markdown 檔案範例：

```md
認識一貫道大綱_凃信埕.md
信愿行大綱_凃信埕.md
修道的好處大綱_凃信埕.md
```

Word 檔案範例：

```md
認識一貫道大綱_凃信埕.docx
信愿行大綱_凃信埕.docx
修道的好處大綱_凃信埕.docx
```

Markdown 大綱使用 `.md`，Word 檔使用 `.docx`，檔名主體一致。

## 十三、檢查規則

完成大綱後，檢查以下項目：

1. 最上方是否標明主題
2. 標號層級是否一致
3. 大標題是否使用「一、二、三」
4. 次標題是否使用「（一）（二）（三）」
5. 若使用第三層，是否使用「1. 2. 3.」
6. 每一層是否有清楚功能
7. 是否有段落順序推進
8. 是否有重複或跳躍
9. 是否過度寫成講稿
10. 是否每段都能收回重點
11. 是否整體主軸清楚
12. 若轉成 Word，是否為標楷體、字體 12、單行距
13. 若要求一張 A4，是否已移除多餘空白行
14. 若要求一張 A4，是否優先精簡為兩層大綱
15. 檔名是否符合「{題目}大綱_凃信埕」
16. 故事是否只保留提示，沒有展開成講稿
17. 提問、比喻、情緒鋪陳是否已留給講稿
18. 收束語是否只保留方向，沒有寫成完整結語
